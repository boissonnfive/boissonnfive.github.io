Je viens de créer un blog sur GitHub grâce à un "module" que j'ai découvert dans l'"Explore" de GitHub : [Simple](http://isnowfy.github.io/simple). Comme son nom l'indique, il est très simple à utiliser. Le blog est semble-t-il hébergé par GitHub, donc gratuitement. Cool. ## Procédure With Simple and github pages you can easily create your static blog. Here is what you need to do. 1. Create a github account. 2. Create a repo named *your_user_name.github.io* (remember to check *Initialize and create README*). 3. Sign in [Simple](http://isnowfy.github.io/simple) with your github account. 4. Click *Initialize* to set up basic files for your static blog site.(maybe a little slow) 5. Click *Go* and start writing. 6. Click *New post* to create a new post and when finish writing click save to generate the static page. 7. Now you can browse the *your_user_name.github.io* site and enjoy it! ## Remarques Avoir écrit la procédure, m'a permis de me familiariser avec le Markdown de ce blog. Il faut obligatoirement donner un nom à la page html contenant l'article du blog que l'on a créé. J'ai eu un problème d'affichage du contenu du billet. Il semble que quand j'ai corrigé les fautes d'orthographe, le contenu du billet se soit affiché. A approfondir, car c'est peut-être dû au fait que je n'avais pas donné un nom à la page HTML et quand je l'ai fait, il a fallu faire une modif dans le contenu du billet. Bref, à voir. Par défaut, quand on arrive sur le blog en mode modifications, il affiche le contenu de notre précédent billet. Mais on n'est pas vraiment dans le billet (post). Il faut bien penser à cliquer sur le nom du billet dans la colonne de gauche, puis faire la modif et enfin cliquer sur *Save*. Enfin, les mots mis en italique en Markdown (entre *) sont soulignés au lieu d'être en **italique**.

Je viens de créer un blog sur GitHub grâce à un "module" que j'ai découvert dans l'"Explore" de GitHub : Simple. Comme son nom l'indique, il est très simple à utiliser. Le blog est semble-t-il hébergé par GitHub, donc gratuitement. Cool.

Procédure

With Simple and github pages you can easily create your static blog. Here is what you need to do.

  1. Create a github account.
  2. Create a repo named your_user_name.github.io (remember to check Initialize and create README).
  3. Sign in Simple with your github account.
  4. Click Initialize to set up basic files for your static blog site.(maybe a little slow)
  5. Click Go and start writing.
  6. Click New post to create a new post and when finish writing click save to generate the static page.
  7. Now you can browse the your_user_name.github.io site and enjoy it!

Remarques

Avoir écrit la procédure, m'a permis de me familiariser avec le Markdown de ce blog.

Il faut obligatoirement donner un nom à la page html contenant l'article du blog que l'on a créé.

J'ai eu un problème d'affichage du contenu du billet. Il semble que quand j'ai corrigé les fautes d'orthographe, le contenu du billet se soit affiché. A approfondir, car c'est peut-être dû au fait que je n'avais pas donné un nom à la page HTML et quand je l'ai fait, il a fallu faire une modif dans le contenu du billet. Bref, à voir.

Par défaut, quand on arrive sur le blog en mode modifications, il affiche le contenu de notre précédent billet. Mais on n'est pas vraiment dans le billet (post). Il faut bien penser à cliquer sur le nom du billet dans la colonne de gauche, puis faire la modif et enfin cliquer sur Save.

Enfin, les mots mis en italique en Markdown (entre *) sont soulignés au lieu d'être en italique.
